# Blaze — user guide

A file-based, git-native issue tracker built for AI coding agents to drive.
This guide takes you from "what is this" to a running board you (or your agent)
operate day to day. Start at the top; each page links onward.

| Page | Read it for |
|---|---|
| [Why Blaze](why-blaze.md) | The problem it solves, who it's for, its strengths, and when *not* to use it. |
| [How it works](how-it-works.md) | The concepts: ticket-is-a-file, directory-is-status, git-is-history, the engine ⟂ data split, and the loops. |
| [Getting started](getting-started.md) | A literal, copy-pasteable walkthrough: install → first ticket → open the board → move it. |
| [Command reference](commands.md) | Every subcommand, flag, and environment variable. |
| [The schema](schema.md) | The default work-item types, fields, and workflows — and how to customise them. |
| [Driving Blaze with an AI agent](driving-with-an-ai-agent.md) | The primary audience: how an agent operates the board with file tools or the CLI. |
